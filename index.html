2. 실시간 고객 정보 제공을 위한 데이터 ETL 구축

역할: 데이터 추출 이후 데이터 로드 전략 수립 및 API 개발

배경(Why)

  • 판매인이 고객의 전화를 수신 시, 구독 및 계약 관련 정보 즉시 확인 가능하도록 개발
  • SAP 등 레거시 시스템에서 수백만 건 데이터를 안정적으로 통합하고 운영 DB에 반영
  • 정보 지연(Latency) 없이 즉시 팝업 제공 필요

접근(How)

  • BigQuery → S3 CSV → DB Load 적재 전략 설계 및 개발
    • 스케줄러(@Scheduled)기반 일일 ETL 수행
  • 테이블 스왑(Table Swap)으로 운영 DB 부하 최소화
  • REST API 설계 및 제공 → 휴대폰에서 고객 정보 즉시 확인 가능
  • 트러블 슈팅
    • 휴대폰 수신 시 알림 정보 1~2초 지연 발생
    • 복합 인덱스 적용으로 조회 지연 해소

성과(Result)

  • 고객 정보 즉시 팝업으로 지연 없이 노출
  • 테이블 스왑 전략과 쿼리 최적화로 대용량 데이터 안정적 갱신 가능
  • Native 앱에서 바로 사용할 수 있는 API 제공으로 사용자 경험 개선
